HTTP

è un protocollo dove viaggiano le interazioni client / server.
il Client è quello che vede l'utente finale. Il server è quello da cui arrivano le informazioni

come funziona questa interazione?

  • HTTP Request

    La richiesta che il client invia al server

  • HTTP Response

    La risposta al client che arriva dal server

Quali sono le informazioni necessarie?

  • URL - https://google.com/apis/test
  • Method
    • GET: è una richiesta per ottenere informazioni dal server. Vado a leggere delle risorse
    • POST: Creo una risorsa che ancora non esiste (Un nuovo ordine su amazon)
    • PUT: Serve per modificare una risorsa esistente (Cambio password / Messaggio da modificare sui social / Cambiare il carrello di amazon)
    • PATCH
    • DELETE: Come dice il nome stesso, serve per eliminare una risorsa esistente
  • Headers - Authentication(richieste di autenticazioni), cookies, Content-Type
  • Payload - data o empty: carico di dati da allegare alla richiesta,

    Si invia solo su POST e PUT

HTTP Response

Cosa ritorna dal server?
  • Payload contiene le informazioni richieste dalla request
  • Status Code fornisce il risultato dell'operazione
  • Headers - meta-informazioni aggiuntive
